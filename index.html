Introduction

Javascript, also known as JS, is a cross-platform, object-oriented scripting language that provides interactivity to a website. Users experience Javascript throughout most websites through fluctuating layouts, responses to button clicks, image galleries, and much more.

What you should already know...

This guide assumes you are currently familiar with and have an understanding of the Internet and the World Wide Web (WWW), HyperText Markup Language (HTML), and have prior programming experience.

Be sure to properly organize and have a sensible file structure within the webpage root folder or directory, before utilizing Javascript. Aside from the HTML index file, different external files of code, like those containing the Javascript code or the CSS stylesheets, should be in a separate folder. The external Javascript file should be in a scripts folder, and the Javascript file should be linked appropriately within the index file, like you would for an external CSS stylesheet.

To link an external Javascript file in the index.html file, use the following element within the index.html:

<script type="text/javascript" src="scripts/main.js"></script>

The external Javascript file does not need to be named "main". Common names for Javascript files is "js", which allows the programmer to quickly identify the file. Additionally, the above element can either be placed in the head or the body of the index.html.
JS Basics

Variables

Variables appear in other programming languages. In Javascript, variables are containers that store data type values. These data types include arrays, booleans, null, numbers, objects, strings, and undefined.

To create a variable, the keyword "var", "const", or "let" is used before providing a name for the variable and a semi-colon. To store a value for the variable and equal sign and value is declared before the semi-colon. The keyword used depends on the programming needs.

Beware that JavaScript is case sensitive, so be consistent with your naming conventions. Moreover, there are some naming restrictions. Names you choose must start with a letter, underscore (_), or dollar sign ($). Only subsequent characters in the name can also be digits (0-9).

GOOD PRACTICE TIP: Use camelCase naming conventions!

var name;
var greeting = "hi";
let times = 4;
const numWord = "two";

Declaration vs. Initialization

You can declare a variable in two ways:

With the keyword var. For example, var x = 42. This syntax can be used to declare both local and global variables, depending on the execution context.

With the keyword const or let. For example, let y = 13. This syntax can be used to declare a block-scope local variable. The const keyword creates a block-scoped, read-only variable.

You can also simply assign a value to a variable For example, x = 42. This form creates an undeclared global variable. It also generates a strict JavaScript warning. Undeclared global variables can often lead to unexpected behavior. Thus, it is discouraged to use undeclared global variables.

A variable declared using the var or let statement with no assigned value specified has the value of undefined. An attempt to access an undeclared variable results in a ReferenceError exception being thrown. You can use undefined to determine whether a variable has a value. The undefined value behaves as false when used in a boolean context. The undefined value converts to NaN (meaning Not a Number) when used in numeric context. When you evaluate a null variable, the null value behaves as 0 in numeric contexts and as false in boolean contexts.

In simplified terms, declaration names and creates the variable (allocates storage for the variable within a corresponding scope), while initialization (which happens automatically when a variable is declared) allows for storage of a value in the variable (be it undefined by default or another data type). Assignment is essentially when a specific value is provided to the variable.

GOOD PRACTICE TIP: Place all declarations at the top of each function or script. The same should be done for loop variables as well. Initializing the variables when you declared them is also good coding practice. Initializing variables provides an idea of the intended use (and intended data type).

let x; // Declaration and initialization
x = "Hello World"; // Assignment
// Or all in one
let y = "Hello World";

Scope

In JavaScript there are two types of scope, local and global scope. JavaScript has function scope: Each function creates a new scope. Scope determines the accessibility (visibility) of variables. In JavaScript, objects and functions are also variables.

Variables defined inside a function are not accessible (visible) from outside the function.

Variables declared within a JavaScript function, become LOCAL to the function. Local variables have Function scope, meaning they can only be accessed from within the function. Since local variables are only recognized inside their functions, variables with the same name can be used in different functions. Local variables are created when a function starts, and deleted when the function is completed.

// code here can NOT use carName
function myFunction() {
var carName = "Volvo";
// code here CAN use carName
}

A variable declared outside a function, becomes GLOBAL. A global variable has global scope, meaning all scripts and functions on a web page can access this variable.

var carName = "Volvo";
// code here can use carName
function myFunction() {
// code here can also use carName
}

If you assign a value to a variable that has not been declared, it will automatically become a GLOBAL variable. In a web browser, global variables are deleted when you close the browser window (or tab), but remain available to new pages loaded into the same window.

myFunction();
// code here can use carName
function myFunction() {
carName = "Volvo";
}

With JavaScript, the global scope is the complete JavaScript environment. Unlike in HTML, when the global scope is the window object and all global variables belong to the window object.

GOOD PRACTICE TIP: Do NOT create global variables unless you intend to. Your global variables (or functions) can overwrite window variables (or functions). Any function, including the window object, can overwrite your global variables and functions.

Hoisting

Hoisting is JavaScript's default behavior of moving declarations to the top of the current scope (to the top of the current script or the current function). In JavaScript, a variable can be declared after it has been used, meaning a variable can be used before it has been declared. Variables and constants declared with let or const are not hoisted! Moreover, JavaScript only hoists declarations, not initializations.

x = 5; // Assign 5 to x
elem = document.getElementById("demo"); // Find an element
elem.innerHTML = x; // Display x in the element
var x; // Declare x

is the SAME as

var x; // Declare x
x = 5; // Assign 5 to x
elem = document.getElementById("demo"); // Find an element
elem.innerHTML = x; // Display x in the element

BUT

var x = 5; // Initialize x
var y = 7; // Initialize y
elem = document.getElementById("demo"); // Find an element
elem.innerHTML = x + " " + y; // Display x and y

is NOT the same as

var x = 5; // Initialize x
elem = document.getElementById("demo"); // Find an element
elem.innerHTML = x + " " + y; // Display x and y
var y = 7; // Initialize y

This is because only the declaration (var y), not the initialization (=7) is hoisted to the top.

Because of hoisting, y has been declared before it is used, but because initializations are not hoisted, the value of y is undefined. That is like writing,

var x = 5; // Initialize x
var y; // Declare y
elem = document.getElementById("demo"); // Find an element
elem.innerHTML = x + " " + y; // Display x and y
y = 7; // Assign 7 to y

Functions

A function is a JavaScript procedure—a set of statements that performs a task or calculates a value. To use a function, you must define it somewhere in the scope from which you wish to call it. A function definition (also called a function declaration, or function statement) consists of the function keyword, followed by the name of the function, a list of parameters to the function (which is enclosed in parentheses and separated by commas), and JavaScript statements that define the function, enclosed in curly brackets, { }.

Function parameters are the names listed in the function definition. Function arguments are the real values passed to (and received by) the function. A function can be given multiple parameters and arguments, which are separated by commas within the parentheses. The parameters, in a function call, are the function's arguments. If a function is called with missing arguments (less than declared), the missing values are set to undefined.JavaScript arguments are passed by value, meaning the function only gets to know the values, not the argument's locations.If a function changes an argument's value, it does not change the parameter's original value. Changes to arguments are not visible or reflected outside the function. The arguments of a function are not limited to strings and numbers. You can pass whole objects to a function.

function functionName(parameter1, parameter2, parameter3) {
// code to be executed
}

The function square takes one parameter, called number. The function consists of one statement that says to return the parameter of the function (that is, number) multiplied by itself. The statement return specifies the value returned by the function.

function square(number) {
return number * number;
}

Parameters, such as a number, are passed to functions by value. The value is passed to the function, but if the function changes the value of the parameter, this change is not reflected globally or in the calling function.

If you pass an object, such as Array or a user-defined object, as a parameter and the function changes the object's properties, that change is visible outside the function (as shown below).

function myFunc(theObject) {
theObject.make = 'Toyota';
}
var mycar = {make: 'Honda', model: 'Accord', year: 1998};
var x, y;
x = mycar.make; // x gets the value "Honda"
myFunc(mycar);
y = mycar.make; // y gets the value "Toyota"
// (the make property was changed by the function)

Functions do not require a name.

function square(number) {
return number * number;
}

could instead have been written as:

var square = function(number) { return number * number; };

where, when either function is used, returns the same value.

return square(4);// returns the value 16
var x = square(4); // x gets the value 16

A method is a function that is a property of an object.

Defining a function does not execute it. Defining the function simply names the function and specifies what to do when the function is called. Calling the function actually performs the specified actions with the indicated parameters.

return square(4);// calls the function and returns the value 16

The preceding statement calls the function with an argument of 4. The function executes its statements and returns the value 16. Functions must be in scope when they are called, but the function declaration can be hoisted.

Variables defined inside a function cannot be accessed from anywhere outside the function, because the variable is defined only in the scope of the function. However, a function can access all variables and functions defined inside the scope in which it is defined. In other words, a function defined in the global scope can access all variables defined in the global scope. A function defined inside another function can also access all variables defined in its parent function and any other variable to which the parent function has access.

// The following variables are defined in the global scope
var num1 = 20,
num2 = 3,
name = 'Chamahk';
// This function is defined in the global scope
function multiply() {
return num1 * num2;
}
multiply(); // Returns 60
// A nested function example
function getScore() {
var num1 = 2,
num2 = 3;
function add() {
return name + ' scored ' + (num1 + num2);
}
return add();
}
getScore(); // Returns "Chamahk scored 5"

A function can refer to and call itself. There are three ways for a function to refer to itself, which are by using the function's name, arguments.callee, or by using an in-scope variable that refers to the function.

Events

These are code structures which listen for things happening in browser, running code in response.The most obvious example is the click event, which is fired by the browser when you click on something with your mouse.

document.querySelector('html').onclick =
function() {
alert('Ouch! Stop poking me!');
}

The above code entered into the console or text editor causes an alert pop-up message window to appear when the mouse is clicked. Moreover, the pop-up window would state the page says the given string, "Ouch! Stop poking me!".

Glossary

Array - a data type. This structure that allows for multiple stored values in one single reference

Example:

var myVariable = [1,'Bob','Steve',10];

Refer to each member of the array using an indexed number in square brackets for the specified array element (starting at [0] for the first positioned element) like this:

myVariable[0], myVariable[1], etc.

Assignment - a specified value is given to a variable

Example:

x = "Hello World";

Boolean - a data type. A True/False value, which are special keywords in JS, and don't require quotes around the value

Example:

var myVariable = true;

camelCase - a naming convention in which

Examples: Num_hits, numHits, temp99, $cred, _name

Declaration - registers an identifier name and corresponding scope to a variable

Function - a set of statements that performs a task

Example:

var factorial = function fac(n) { return n < 2 ? 1 : n * fac(n - 1); };
console.log(factorial(3));

Hoisting - JavaScript variables are hoisted, meaning that within a variable’s current scope, the variable will be hoisted to the top regardless of its placement. If the variable is initialized, when it is hoisted to the top it will initially be set to undefined.

Initialization - memory is allocated to the variable

Number - a data type. A numerical value that does not require quotes around the value

Example:

var myVariable = 10;

Object - a data type and basically, any element. Everything in JavaScript is an object, and can be stored in a variable

Example:

var myVariable = document.querySelector('h1');

Scope - the accessibility (visibility) of variables for use

String - a data type. A sequence of text enclosed in single or double quote marks

Example:

var myVariable = 'Bob';

Variable - container that stores data value(s), which can be changed at any time

Example:

var greeting = "hi";
